To ensure a seamless and responsive user experience, the frontend utilizes Express view generators (EJS) for dynamic rendering and Bootstrap for the UI layout.⚙️ How to Run the Application (Setup Instructions)Follow these steps to run MediSync locally on your machine:Clone the repository:git clone [https://github.com/MuhammadUsman3063/Advance-Web-Assignment-2.git](https://github.com/MuhammadUsman3063/Advance-Web-Assignment-2.git)

The server will start running at http://localhost:5000.🚀 Lab Mid Criteria MappingsSection 1: API Design & Functionality (Criteria 1)The application strictly follows RESTful principles to manage resources, utilizing standard HTTP methods and proper JSON responses with appropriate status codes (e.g., 200 OK, 201 Created, 400 Bad Request, 500 Server Error).POST /api/patients/register – Creates a new patient record and assigns a sequential token (Status: 201).GET /api/patients – Fetches the active waiting queue (Status: 200).PUT /api/patients/next – Updates the status of the current patient to 'Completed' and calls the next pending patient (Status: 200).DELETE /api/patients/:id – Removes a patient from the queue if they cancel or no-show (Status: 200).Section 2: Database Integration (Criteria 2)The backend integrates seamlessly with MongoDB using Mongoose as the Object Data Modeling (ODM) library.Patient Schema: Contains fields for name (String, required), phone (String, required), tokenNumber (Number), and status (Enum: 'Pending', 'Serving', 'Completed').User Schema: Stores clinic staff details with roles (e.g., 'Doctor', 'Receptionist') and encrypted passwords.Section 3: Express Generator & EJS (Criteria 3)The project structure was initialized using the Express application generator, separating concerns into routes/, views/, and public/ directories.Dynamic Views: EJS (Embedded JavaScript) is utilized to dynamically render pages server-side.Data Passing: The Express controllers fetch data from MongoDB and pass it directly to the EJS templates using res.render().Section 4: Bootstrap & UI Design (Criteria 4)The user interface is fully responsive and mobile-friendly, implemented entirely using the Bootstrap 5 framework.Layout: Utilizes the Bootstrap Grid system (containers, rows, columns).Components: Bootstrap Cards for analytics, Tables for the active queue, and styled Buttons for user actions.Section 5: Code Quality & Organization (Criteria 5)The codebase follows the MVC (Model-View-Controller) architecture to maintain high readability and modularity.Code is logically separated into models, controllers, routes, and middleware directories.Custom middleware functions are utilized for error handling and protecting secure routes.Section 6: Extra Features / Creativity (Criteria 6)To elevate the application beyond basic CRUD operations:Authentication: JWT (JSON Web Token) based authentication ensures secure staff login.Search & Filtering: The history panel includes search capabilities.Real-Time Capabilities: Implemented foundational logic for live queue updates.Developed with ❤️ by Usman for Advanced Web Technologies Lab Mid Term.
